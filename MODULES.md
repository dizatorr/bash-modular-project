## Общая информация

### Структура модуля
Каждый модуль имеет следующую структуру метаданных:
```
#!/usr/bin/env bash
# === MENU: Название для меню
# === FUNC: имя_функции
# Автор: Имя Автора
# Версия: X.X
# Лицензия: MIT
# Описание: Подробное описание модуля
#
# === Список функций ===
# - имя_функции() - описание функции
# - другая_функция() - описание другой функции
#
# === Требования ===
# - Внешние зависимости: зависимости
# - Зависимости от других модулей: зависимости
# - Права доступа: права
# - Требуемые переменные окружения: переменные
#
# === Примеры использования ===
# - вызов функции
```

### Категории модулей
- **Сетевая диагностика:** Модули для анализа, мониторинга и диагностики сетевых соединений
- **SMB ресурсы:** Модули для подключения, монтирования и управления SMB/CIFS ресурсами
- **Вспомогательные:** Примеры и базовые функции для демонстрации работы системы
- **Специализированные:** Модули для специфических задач, таких как настройка SSH или Ansible

### Зависимости между модулями
Некоторые модули зависят от функций, определенных в других модулях:
- `net_select_interface` - используется модулями: network_traffic_monitor, setup_dhcp_server, scan_network_devices, network_interface_stats
- `smb_load_resources`, `smb_get_display_names`, `smb_get_username` - используются модулями: smb_mount_resource

### Внешние зависимости
Некоторые модули требуют установки дополнительных пакетов:
- Для работы с сетью: `ping`, `nslookup`, `ipcalc`, `dnsmasq`, `ip`, `nmcli`
- Для работы с SMB: `smbclient`, `cifs-utils`
- Для мониторинга трафика: `iftop`
